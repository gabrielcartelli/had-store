Trabalho Avaliativo - Hat Store

Considerações Gerais

  • O trabalho deve ser realizado em grupo (conforme separação do discord).
  • A aplicação a ser testada é a Hat Store (hatstore-prd.fly.dev).
  • A documentação da aplicação está disponível no rodapé do site (hatstore-prd.fly.dev/documentacao.html).
  • Não é permitido utilizar inteligência artificial para realizar o trabalho, apenas para consulta de dúvidas e informações.
  • Não é permitido copiar informações de trabalhos de outros grupos.
  • O Hat Store é uma loja fictícia, portanto nenhuma cobrança ou compra real será efetuada.
  • A entrega do trabalho deve ser feita até 15/09/2025 às 23:59:59 (horário de Brasília). Entregas após esse prazo serão desconsideradas.
  • Cada grupo receberá um feedback referente ao trabalho entregue.

Objetivos

  1. Refinar e estimar as histórias das seguintes novas funcionalidades:
    • Controle de estoque
    • Filtro de produtos por categoria e faixa de preço
    Considerações:
    • O refinamento e as estimativas serão feitas em aula em dinâmica
  2. Escrever os casos de teste das novas funcionalidades que foram refinadas
    Considerações:
    • Os casos de teste devem ser registrados no Test Plan e escritos em Gherkin.
    • Em cada caso de teste deve ser indicado o Nível de Teste (Aceitação, Sistema, Integração ou Unitário).
    • Lembre-se de pensar bem e contemplar todos os casos possíveis que forem pertinentes, considerando fluxos felizes e negativos.
    • Dessa vez os casos de teste serão escritos tendo como base somente as histórias, exercitando assim a prática de planejar os testes antes da codificação ser finalizada
    • O prazo para finalizar essa etapa é 12/09/2025 às 13:29:59 (horário de Brasília)
  3. Revisão de código backend
    Considerações:
    • Durante a manhã de sexta-feira (12/09/2025), será disponibilizado o pull request com o código do backend para revisão.
    • Cada grupo deve revisar o código e sugerir melhorias e correções no privado (discord ou teams) para o instrutor Gabriel Cartelli.
    • O prazo para finalizar essa etapa é 12/09/2025 às 13:29:59 (horário de Brasília)
  4. Executar os casos de teste que foram escritos (parte 1).
    Considerações:
    • No fim da aula de sexta-feira (12/09/2025), será feito o deploy do backend e os testes devem ser iniciados.
    • O frontend só será disponibilizado na segunda-feira (15/09/2025), então o tempo para executar os testes tendo o front será curto, por isso cada grupo deve montar uma estratégia de execução dos testes para validar o máximo possível das funcionalidades diretamente pela API.
    • Nessa etapa problemas encontrados já devem ser reportados no Azure.
    • Lembre-se de registrar evidências durante a execução dos testes.
  5. Revisão de código frontend
    Considerações:
    • Durante a tarde de sexta-feira (12/09/2025), será disponibilizado o pull request com o código do frontend para revisão.
    • Cada grupo deve revisar o código e sugerir melhorias e correções no privado (discord ou teams) para o instrutor Gabriel Cartelli.
    • O prazo para finalizar essa etapa é 15/09/2025 às 08:30:00 (horário de Brasília)
  6. Executar os casos de teste que foram escritos (parte 2).
    Considerações:
    • No início da manhã de segunda-feira (15/09/2025), será feito o deploy do frontend e os testes devem agora ser finalizados considerando também o frontend.
    • Os defeitos encontrados devem ser registrados no Azure.
    • Lembre-se de registrar evidências durante a execução dos testes.
  7. Relatório com parecer e métricas
    • No relatório deve estar incluso a resposta da pergunta:
      Esse pacote de alterações do Hat Store com novas funcionalidades pode ser publicado no ambiente produtivo? Justifique sua resposta.
    • Também deve incluir métricas que forem pertinentes conforme aula ministrada na sexta-feira (12/09/2025)
    Considerações:
    • Este relatório deve ser entregue no formato de PDF no seguinte repositório: GitLab.
    • Somente uma pessoa do grupo precisa fazer o upload do relatório.
    • Lembre-se de identificar o relatório com o número do grupo e montar um documento minimamente estruturado pensando que analistas de negócio e outras pessoas de interesse irão lê-lo.
    • O prazo limite para entregar o trabalho de forma completa com essa última etapa é 15/09/2025 às 23:59:00 (horário de Brasília)
