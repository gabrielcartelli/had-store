Trabalho Avaliativo - Hat Store

Considerações Gerais

  • O trabalho deve ser realizado individualmente.
  • A aplicação a ser testada é a Hat Store (hat-store-training.fly.dev).
  • A documentação da aplicação está disponível no rodapé do site (hat-store-training.fly.dev/documentacao.html).
  • Não é permitido utilizar inteligência artificial para realizar o trabalho, apenas para consulta de dúvidas e informações.
  • Não é permitido copiar informações de trabalhos de outros Cresceres.
  • O Hat Store é uma loja fictícia, portanto nenhuma cobrança ou compra real será efetuada.
  • A entrega do trabalho deve ser feita até 08/09/2025 às 23:59:59 (horário de Brasília). Entregas após esse prazo serão desconsideradas.
  • Cada Crescer receberá um feedback referente ao trabalho entregue.

Objetivos

  1. Escrever os casos de teste dos seguintes fluxos em gherkin:
    • Listagem e pesquisa de chapéus;
    • Fluxo de compra de chapéus;
    • Funcionalidade de cupom de desconto.
    Considerações:
    • Não é necessário incluir os fluxos de login e consulta de pedidos.
    • Os casos de teste devem ser registrados no Test Plan.
    • Em cada caso de teste deve ser indicado o Nível de Teste (Aceitação, Sistema, Integração ou Unitário).
    • Em cada caso de teste deve ser indicada a Técnica de Teste utilizada, conforme a tabela abaixo:
      • Técnicas de caixa-preta: Particionamento de Equivalência (EP), Análise de Valor Limite (BVA), Tabela de Decisão, Transição de Estados, Pairwise Testing (Combinatorial).
      • Técnicas baseadas na experiência: Testes Exploratórios, Testes Baseados em Erro, Checklists/Definições de Comportamento.
  2. Executar os casos de teste que foram escritos.
    Considerações:
    • Cada caso de teste deve ser executado pelo runner do Test Plan.
    • Cada caso de teste deve ter pelo menos uma evidência comprovando que foi executado com sucesso.
    • Os casos de teste que falharem devem ser registrados com status de falha.
  3. Registrar os defeitos encontrados durante os testes.
    Considerações:
    • Os defeitos devem ser registrados no board do Azure (dev.azure.com/cwi-formacao) na user story do seu nome.
    • Os defeitos devem seguir o padrão apresentado em aula.
  4. Considerando que esses testes foram feitos para avaliar o desenvolvimento de novas funcionalidades no site Hat Store, responda à seguinte pergunta:
    Esse pacote de alterações do Hat Store com novas funcionalidades pode ser publicado no ambiente produtivo? Justifique sua resposta.
    Considerações:
    • A resposta dessa pergunta deve ser entregue no GitLab.
